# 🎨 نش گرافیک (Nash Graphic) — راهنمای پروژه

پلتفرم فارسی‌زبان طراحی گرافیک و چاپ — بازارچه طرح، اتصال طراح/چاپخانه/مشتری، و ابزارهای هوشمند محتوا.

## ساختار فایل‌ها

```
nash-graphic/
├── index.html                 ← صفحه اصلی (HTML)
├── supabase-schema.sql        ← اسکیمای پایه دیتابیس
├── migration-order-flow.sql   ← migration فرآیند سفارش
├── css/
│   └── styles.css             ← تم "Liquid Dark Premium" + ۳ حالت رنگی
└── js/
    ├── config.js       ← اتصال Supabase + دیکشنری i18n (فارسی/انگلیسی) + Router
    ├── auth.js         ← ثبت‌نام/ورود/خروج + پروفایل + تیکت پشتیبانی + پیگیری سفارش
    ├── cart.js         ← سبد خرید + پکیج‌ها + ثبت سفارش (بدون درگاه پرداخت آنلاین)
    ├── marketplace.js  ← بازارچه طرح + آپلود + نظرات + امتیاز + علاقه‌مندی
    ├── dashboard.js     ← داشبورد طراح و داشبورد چاپخانه (دسته‌بندی، سفارش، پکیج)
    ├── admin.js         ← پنل ادمین کامل (کاربران، سفارش‌ها، محتوا، تنظیمات، تم، رسانه)
    ├── portfolio.js     ← پورتفولیوی عمومی طراحان (نمونه‌کار، رزومه، دسته‌بندی)
    ├── app.js           ← راه‌اندازی کلی + هدر/منو + صفحه خانه + وبلاگ + FAQ + قوانین
    └── nash-agents.js   ← ابزارهای هوشمند: Content Agent (بلاگ خودکار) + Support Agent (چت پشتیبانی)
```

---

## 🧠 ابزارهای هوشمند (Nash Agents)

فایل `js/nash-agents.js` دو ابزار مستقل اضافه می‌کند:

**۱. Content Agent** (دکمه آبی، فقط برای ادمین)
سایت منبع را اسکن می‌کند، مقالات را با **Google Gemini API** (رایگان) ترجمه/بازنویسی/خلاصه می‌کند و مستقیم در جدول `blog` ذخیره می‌کند.

**۲. Support Agent** (دکمه طلایی، برای همه کاربران)
چت پشتیبانی هوشمند که با Gemini AI به سوالات پاسخ می‌دهد؛ اگر AI در دسترس نبود، به یک درخت تصمیم از پیش تعریف‌شده برمی‌گردد.

**تنظیم کلید Gemini:** پنل ادمین → تب «رسانه» → بخش «کلید Gemini API» (رایگان از [aistudio.google.com](https://aistudio.google.com)).

## 🚀 مراحل نصب

### ۱. ساخت پروژه Supabase
1. به [supabase.com](https://supabase.com) بروید و **New Project** بزنید
2. منطقه **Europe (Frankfurt)** را انتخاب کنید (نزدیک‌ترین به ایران)

### ۲. فعال‌سازی Authentication
**Authentication → Providers** → **Email** را فعال، **Confirm email** را برای تست سریع‌تر غیرفعال کنید.

### ۳. اجرای اسکیمای دیتابیس
در **SQL Editor**، محتوای `supabase-schema.sql` را کامل اجرا کنید — این تمام جدول‌ها، RLS policyها، و bucketهای Storage را می‌سازد.

⚠️ **نکته مهم:** کل فایل را یکجا اجرا کنید، نه بخشی از آن — اجرای ناقص باعث می‌شود جدول‌ها یا ستون‌هایی (مثل `resume`, `portfolio_items`, دسته‌بندی‌های پیش‌فرض) از قلم بیفتند.

### ۴. ساخت کاربر ادمین
در **Authentication → Users → Add User** یک کاربر بسازید، سپس در SQL Editor:
```sql
update public.users set role = 'admin' where email = 'YOUR-EMAIL@example.com';
```

### ۵. کلیدهای Supabase
کلیدها از قبل در `js/config.js` تنظیم شده‌اند. برای پروژه جدید، از **Settings → API** کپی و جایگزین کنید.

### ۶. Deploy
**GitHub Pages (پیشنهادی، رایگان):**
1. تمام فایل‌ها را در ریشه ریپازیتوری آپلود کنید
2. **Settings → Pages → Branch: main** را فعال کنید
3. بعد از هر تغییر، Cache مرورگر را پاک کنید تا نسخه جدید دیده شود

## 🔧 ویژگی‌های فنی

- Supabase Auth + RLS کامل + Storage (۷ باکت)
- Realtime روی سفارش‌ها، نوتیفیکیشن، تیکت‌ها
- دوزبانه فارسی/انگلیسی با تغییر واقعی `dir`/`lang` (سازگار با Google Translate)
- Content Agent و Support Agent مبتنی بر Gemini AI (رایگان)
- بدون بک‌اند اختصاصی — کاملاً static + Supabase
- طراحی Responsive کامل، هدر دو‌ردیفه با منوی همیشه‌قابل‌مشاهده

## ⚠️ نکات مهم برای توسعه بیشتر

1. بعد از هر تغییر در فایل‌های JS/CSS، حتماً **کل پوشه** را جایگزین کنید، نه فایل تکی — تا مطابقت نسخه‌ها حفظ شود
2. بعد از آپلود روی GitHub، چند دقیقه صبر کنید (زمان deploy) و سپس Cache مرورگر را پاک کنید
3. یک خطای Syntax در هر فایل JS، اجرای کل اسکریپت‌های بعد از آن را متوقف می‌کند — همیشه فایل نهایی را با ابزار syntax-check تأیید کنید
4. برای ذخیره‌سازی دارایی‌های سنگین (مثل دنباله فریم ویدیو)، بجای Supabase Storage از مسیر مستقیم در ریپازیتوری (پشت CDN رایگان GitHub Pages) استفاده کنید
